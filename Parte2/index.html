# XML JSON Semejansas
1 Se trata de un formato de intercambio de datos estándar abierto basado en texto. Es un lenguaje de marcado que define la estructura de cualquier archivo XML en una estructura de árbol. Es usado como un formato de intercambio para que los usuarios muevan datos entre aplicaciones similares.
2 Es ligero y fácil de leer,pero no proporciona información sobre el esquema o el tipo. Es legible por los humanos pero no necesariamente por las máquinas. Se procesa facilmente debido a que la estructura de los datos es simple y estandar.
3 Es ideal para compartir datos entre varias aplicaciones. Puede utilizarse para compartir información estructurada entre programas y documentos. Estan compuesto por un formato de texto plano
4 Es una forma más larga de representar datos estructurados. Es una forma ligera y legible para el ser humano de representar estructuras de datos. Tiene una autodesripcion que es legible por el ser humano
5 Puede almacenarse como archivo de texto. Puede utilizarse con archivos JavaScript o de texto plano. Es jerarquico donde esto nos dice que los valores pueden contener listas de objetivos o valores
6 Utiliza mas memoria al momento de procesar la informacion. Utiliza menos memoria a la hora de procesar la informacion.
7 Se utiliza para representar datos de forma legible por la máquina. Se utiliza para almacenar y transmitir datos.

Que es JSON

Es un formato de texto sencillo para el intercambio de datos. Se trata de un subconjunto de la notación literal de objetos de JavaScript, aunque, debido a su amplia adopción como alternativa a XML, se considera un formato independiente del lenguaje.
